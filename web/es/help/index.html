Usar Tutor1on1 según el rol.

Flujo profesor

Publicar y personalizar cursos.

  1. Inicie sesión como profesor o registre una cuenta de profesor.
  2. Cree o cargue una versión local del curso.
  3. Suba el bundle validado al marketplace.
  4. Apruebe solicitudes de inscripción de estudiantes.
  5. Revise en la página del profesor el progreso, errores recurrentes y sesiones sincronizadas de cada estudiante.

Flujo estudiante

Inscribirse y estudiar a su ritmo.

  1. Inicie sesión como estudiante o registre una cuenta de estudiante.
  2. Abra el marketplace y solicite inscripción en un curso.
  3. Espere la aprobación del profesor.
  4. Descargue el bundle aprobado del curso.
  5. Abra el curso e inicie una sesión Learn o Review que siga su progreso.

Ciclo de aprendizaje personalizado

1

Aprender

El tutor explica el siguiente concepto usando el contenido del curso y el progreso actual del estudiante.

2

Practicar

Los ejercicios se ajustan a las respuestas del estudiante, con pistas, ejemplos más simples y feedback positivo cuando hace falta.

3

Repasar errores

Los errores importantes se resumen en el cuaderno de errores para que el repaso apunte a debilidades reales.

Campos de API config

  • Provider elige el servicio de modelos, como OpenAI o SiliconFlow.
  • Base URL es el endpoint que Tutor1on1 usa para el proveedor seleccionado.
  • API Key se guarda localmente y se usa para tutor, TTS y STT cuando esas funciones lo necesitan.
  • Test API key llama a {baseUrl}/models, verifica la key y carga las listas de modelos text, TTS y STT disponibles. No descarga modelos ni selecciona automáticamente un modelo nuevo.
  • Text model alimenta la conversación del tutor. Thinking effort aparece solo si el provider admite controles de reasoning.
  • TTS model controla la salida de voz. STT model controla voz a texto. Pueden quedar vacíos si no usa voz.
  • Saved API configs guarda provider URL, modelos seleccionados, thinking effort y key hash para reutilizar configuraciones verificadas.
Marketplace

Descubrir cursos

Los profesores publican cursos. Los estudiantes exploran, solicitan inscripción y descargan después de la aprobación.

Sessions

Feedback personal

Learn sirve para material nuevo y Review para práctica de recuerdo. El feedback se adapta a la respuesta del estudiante y mantiene visible el progreso.

Sync

Continuidad

Las páginas de profesor y estudiante sincronizan inscripciones, sesiones, progreso y señales del cuaderno de errores vinculadas a la cuenta.

Problemas comunes

  • Si falla el inicio de sesión o la carga del curso, compruebe que la app llegue a api.tutor1on1.org.
  • Si un estudiante no puede entrar a un curso, confirme que la inscripción fue aprobada y que el bundle se descargó.
  • Si la lista de modelos parece antigua, ejecute Test API key otra vez para el provider y key activos.
  • Si el progreso parece desactualizado entre dispositivos, espere a que termine la sincronización o ejecute una actualización manual.
