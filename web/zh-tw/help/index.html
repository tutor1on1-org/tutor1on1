快速開始

按角色使用 Tutor1on1。

老師發布並管理課程；學生申請報名、下載已批准內容，並獲得符合個人進度的指導、練習和錯題複習。

先安裝 老師流程 學生流程 API config

老師流程

發布並個人化管理課程。

  1. 以老師身份登入，或註冊老師帳號。
  2. 建立或載入本機課程版本。
  3. 將已校驗的 bundle 上傳到 marketplace。
  4. 審批學生報名請求。
  5. 在老師首頁查看每位學生的進度、常見錯誤和同步過來的會話。

學生流程

報名並按自己的節奏學習。

  1. 以學生身份登入，或註冊學生帳號。
  2. 打開 marketplace，申請報名課程。
  3. 等待老師批准。
  4. 下載已批准的課程 bundle。
  5. 打開課程，開始跟隨個人進度的 Learn 或 Review 會話。

個人化學習閉環

1

學習

Tutor 會結合課程內容和學生目前進度，講解下一步知識點。

2

練習

練習會根據學生回答調整，必要時給提示、更簡單的例子和正向回饋。

3

複習錯題

有意義的錯誤會進入錯題集，讓後續複習真正針對薄弱點。

API config 內的內容

  • Provider 選擇模型服務，例如 OpenAI 或 SiliconFlow。
  • Base URL 是 Tutor1on1 請求模型服務的地址，應用會按目前提供方顯示。
  • API Key 保存在本機，用於 tutor、TTS、STT 等需要模型服務的功能。
  • Test API key 會請求 {baseUrl}/models，驗證 key，並載入可用的文字、TTS、STT 模型列表。它不會下載模型，也不會自動選擇新模型。
  • Text model 用於對話式 tutor。Thinking effort 只在提供方支援 reasoning 控制時顯示。
  • TTS model 控制語音輸出；STT model 控制語音轉文字。如果不用語音，可以留空。
  • 已儲存的 API config 保存 provider URL、選擇的模型、thinking effort 和 key hash，方便重用已驗證配置。
Marketplace

課程發現

老師發布課程展示；學生瀏覽課程、申請報名，並在批准後下載。

Sessions

個人回饋

Learn 用於新內容，Review 用於回憶練習。回饋會根據學生回答調整，並保持進步可見。

Sync

連續學習

老師和學生首頁會同步已連接帳號的報名、會話、進度和錯題信號。

常見問題

  • 如果登入或課程載入失敗，先確認應用能連到 api.tutor1on1.org。
  • 如果學生無法進入課程，先確認報名已批准，並且 bundle 已下載。
  • 如果模型列表看起來過期，對目前 provider 和 key 重新執行「Test API key」。
  • 如果多裝置進度不同步，等待同步完成，或手動刷新一次。
