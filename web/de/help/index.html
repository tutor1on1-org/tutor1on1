Schnellstart und API config

Installieren, als Lehrer oder Schüler anmelden und verstehen, was jedes config-Feld steuert.

Lehrer erstellen oder laden Kurse, laden validierte Bundles hoch, genehmigen Anmeldungen und prüfen Fortschritt sowie Fehlermuster.

Schüler lernen in genehmigten Kursen mit Learn und Review, erhalten passende Übungen und nutzen das Fehlerheft.

API config

  • Provider: Voreinstellung für den API-Dienst, zum Beispiel OpenAI oder SiliconFlow.
  • Base URL: OpenAI-kompatibler endpoint. Test API key ruft baseUrl + /models auf.
  • API key: Anbieterschlüssel, lokal im secure storage für diese Base URL gespeichert.
  • Text model: Tutor-Dialog, Anleitung, Feedback und Review-Bewertung.
  • TTS/STT models: optionale Sprach- und Erkennungsmodelle.
  • Reasoning effort: optionale Einstellung für unterstützte Anbieter.
  • Test API key: prüft Zugriff und aktualisiert Modelllisten. Es lädt keine Modelle herunter und wählt kein neues Modell automatisch aus.

Hilfe

Wenn die Modellliste alt wirkt, Test API key erneut ausführen. Wenn ein Schüler nicht in den Kurs kommt, Genehmigung und Bundle-Download prüfen.
