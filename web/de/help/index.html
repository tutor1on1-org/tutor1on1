Schnellstart

Tutor1on1 nach Rolle verwenden.

Lehrer veröffentlichen und verwalten Kurse. Schüler beantragen die Einschreibung, laden freigegebene Inhalte herunter und erhalten Anleitung, Übungen und Fehlerwiederholung passend zu ihrem Fortschritt.

Zuerst installieren Lehrerablauf Schülerablauf API config

Lehrerablauf

Kurse veröffentlichen und personalisieren.

  1. Als Lehrer anmelden oder ein Lehrerkonto registrieren.
  2. Eine lokale Kursversion erstellen oder laden.
  3. Das validierte Bundle in den Marketplace hochladen.
  4. Einschreibungsanfragen von Schülern genehmigen.
  5. Auf der Lehrer-Startseite Fortschritt, wiederkehrende Fehler und synchronisierte Sitzungen je Schüler prüfen.

Schülerablauf

Einschreiben und im eigenen Tempo lernen.

  1. Als Schüler anmelden oder ein Schülerkonto registrieren.
  2. Den Marketplace öffnen und Einschreibung in einen Kurs beantragen.
  3. Auf die Genehmigung durch den Lehrer warten.
  4. Das genehmigte Kurs-Bundle herunterladen.
  5. Den Kurs öffnen und eine Learn- oder Review-Sitzung starten, die dem eigenen Fortschritt folgt.

Personalisierter Lernkreislauf

1

Lernen

Der Tutor erklärt das nächste Konzept im Kontext des Kurses und des aktuellen Fortschritts des Schülers.

2

Üben

Übungen passen sich den Antworten des Schülers an, mit Hinweisen, einfacheren Beispielen und positivem Feedback bei Bedarf.

3

Fehler wiederholen

Bedeutende Fehler werden im Fehlerheft zusammengefasst, damit spätere Wiederholung echte Schwächen trifft.

API config Felder

  • Provider wählt den Modelldienst, zum Beispiel OpenAI oder SiliconFlow.
  • Base URL ist der Endpoint, den Tutor1on1 für den gewählten Provider nutzt.
  • API Key wird lokal gespeichert und für Tutor, TTS und STT genutzt, wenn diese Funktionen ihn benötigen.
  • Test API key ruft {baseUrl}/models auf, prüft den key und lädt die verfügbaren Text-, TTS- und STT-Modelllisten. Es lädt keine Modelle herunter und wählt kein neues Modell automatisch aus.
  • Text model steuert die Tutor-Unterhaltung. Thinking effort erscheint nur bei Providern, die Reasoning-Kontrollen unterstützen.
  • TTS model steuert Sprachausgabe. STT model steuert Sprache-zu-Text. Beide können leer bleiben, wenn Sprache nicht genutzt wird.
  • Saved API configs speichern Provider URL, gewählte Modelle, thinking effort und key hash, damit geprüfte Setups wiederverwendet werden können.
Marketplace

Kurse entdecken

Lehrer veröffentlichen Kursangebote. Schüler stöbern, beantragen Einschreibung und laden nach Genehmigung herunter.

Sessions

Persönliches Feedback

Learn dient neuem Stoff und Review der Abrufübung. Feedback passt sich der Antwort des Schülers an und macht Fortschritt sichtbar.

Sync

Kontinuität

Lehrer- und Schüler-Startseiten synchronisieren kontoverknüpfte Einschreibungen, Sitzungen, Fortschritt und Fehlerheft-Signale.

Häufige Probleme

  • Wenn Anmeldung oder Kursladen fehlschlägt, prüfen Sie, ob die App api.tutor1on1.org erreicht.
  • Wenn ein Schüler nicht in einen Kurs kommt, prüfen Sie, ob die Einschreibung genehmigt und das Bundle heruntergeladen wurde.
  • Wenn Modelllisten veraltet wirken, führen Sie Test API key erneut für den aktiven Provider und key aus.
  • Wenn Fortschritt auf mehreren Geräten veraltet wirkt, warten Sie auf Sync oder starten Sie eine manuelle Aktualisierung.
