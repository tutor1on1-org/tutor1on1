役割ごとに Tutor1on1 を使う。

先生はコースを公開して管理します。生徒は受講申請を行い、承認済みコンテンツをダウンロードし、進度に合った指導、練習、ミス復習を受けます。

先にインストール 先生の流れ 生徒の流れ API config ChatGPT OAuth

先生の流れ

コースを公開し、継続的に個別化します。

  1. 先生としてログイン、または先生アカウントを登録します。
  2. ローカルのコース版を作成または読み込みます。
  3. 検証済み bundle を marketplace にアップロードします。
  4. 生徒の受講申請を承認します。
  5. 先生ホームで各生徒の進度、よくあるミス、同期済みセッションを確認します。

生徒の流れ

受講し、自分のペースで学びます。

  1. 生徒としてログイン、または生徒アカウントを登録します。
  2. marketplace を開き、コースの受講を申請します。
  3. 先生の承認を待ちます。
  4. 承認済みコース bundle をダウンロードします。
  5. コースを開き、進度に沿った Learn または Review セッションを始めます。

個別学習の流れ

1

学ぶ

Tutor はコース内容と生徒の現在の進度に合わせて、次の概念を説明します。

2

練習する

練習は生徒の回答に合わせて変わり、必要なときはヒント、より簡単な例、前向きなフィードバックを返します。

3

ミスを復習する

意味のあるミスはミスノートに要約され、後の復習が本当の弱点に向かいます。

API config の項目

  • Provider は OpenAI や SiliconFlow などのモデルサービスを選びます。
  • Base URL は Tutor1on1 がモデルサービスへ送信する endpoint です。選択中の provider に合わせて表示されます。
  • API Key はローカルに保存され、tutor、TTS、STT などモデルサービスが必要な機能で使われます。
  • Test API key は {baseUrl}/models を呼び出して key を検証し、利用可能な text、TTS、STT モデル一覧を読み込みます。モデルはダウンロードせず、新しいモデルを自動選択しません。
  • Text model は tutor 会話に使われます。Thinking effort は reasoning control をサポートする provider でのみ表示されます。
  • TTS model は音声出力、STT model は音声入力の文字起こしを制御します。音声を使わない場合は空欄で構いません。
  • 保存済み API config には provider URL、選択済みモデル、thinking effort、key hash が保存され、検証済み設定を再利用できます。

API key の代わりに ChatGPT OAuth を使う

  1. Settings の API Settings を開き、Provider で OpenAI Codex (ChatGPT OAuth) を選びます。
  2. Login with ChatGPT を押し、ブラウザでログインを完了します。自動で戻らない場合は、最後の redirect URL または authorization code をダイアログに貼り付けます。
  3. Refresh models を押し、gpt-5.5 などの text model を選んで API config を保存します。
  4. tutor セッションを開始または開き直し、モデル選択で保存済み Codex OAuth モデルを選びます。

この設定は tutor のテキスト会話に ChatGPT/Codex OAuth quota を使います。音声が必要な場合、TTS と STT は引き続き API-key provider を使います。

Marketplace

コースを見つける

先生はコース listing を公開します。生徒は閲覧、受講申請、承認後のダウンロードを行います。

Sessions

個別フィードバック

Learn は新しい内容、Review は想起練習に使います。フィードバックは生徒の回答に合わせて変わり、進歩を見える形にします。

Sync

継続性

先生と生徒のホームは、アカウントに紐づく受講状態、セッション、進度、ミスノート信号を同期します。

よくある問題

  • ログインまたはコース読み込みに失敗する場合は、アプリが api.tutor1on1.org に接続できるか確認します。
  • 生徒がコースに入れない場合は、受講申請が承認され、bundle がダウンロード済みか確認します。
  • モデル一覧が古く見える場合は、現在の provider と key で Test API key を再実行します。
  • 複数端末の進度が古い場合は、同期完了を待つか手動更新します。
