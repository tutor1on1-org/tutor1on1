クイックスタートと API config

アプリをインストールし、先生または生徒としてログインし、config の各項目を理解します。

先生はコースを作成または読み込み、検証済み bundle をアップロードし、受講申請を承認し、生徒ごとの進度とミス傾向を確認します。

生徒は承認されたコースで Learn と Review を進め、適応練習を受け、ミスノートで弱点を復習します。

API config

  • Provider: OpenAI や SiliconFlow などの API サービス設定。
  • Base URL: OpenAI-compatible endpoint。Test API key は baseUrl + /models を呼び出します。
  • API key: provider key。base URL ごとにローカル secure storage に保存します。
  • Text model: 会話、説明、フィードバック、Review 採点に使うモデル。
  • TTS/STT models: 任意の音声出力と音声認識モデル。
  • Reasoning effort: 対応 provider 向けの任意設定。
  • Test API key: アクセスを検証しモデル一覧を更新します。モデルをダウンロードせず、新モデルを自動選択しません。

ヘルプ

モデル一覧が古い場合は Test API key を再実行します。生徒がコースに入れない場合は承認と bundle ダウンロードを確認します。
