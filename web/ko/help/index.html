역할에 맞게 Tutor1on1 사용하기.

교사는 코스를 공개하고 관리합니다. 학생은 수강을 신청하고 승인된 콘텐츠를 다운로드한 뒤, 자신의 진도에 맞는 지도, 연습, 오답 복습을 받습니다.

먼저 설치 교사 흐름 학생 흐름 API config

교사 흐름

코스를 공개하고 계속 개인화합니다.

  1. 교사로 로그인하거나 교사 계정을 등록합니다.
  2. 로컬 코스 버전을 만들거나 불러옵니다.
  3. 검증된 bundle을 marketplace에 업로드합니다.
  4. 학생 수강 신청을 승인합니다.
  5. 교사 홈에서 학생별 진도, 반복 오답, 동기화된 세션을 확인합니다.

학생 흐름

수강하고 자신의 속도로 공부합니다.

  1. 학생으로 로그인하거나 학생 계정을 등록합니다.
  2. marketplace를 열고 코스 수강을 신청합니다.
  3. 교사 승인을 기다립니다.
  4. 승인된 코스 bundle을 다운로드합니다.
  5. 코스를 열고 진도에 맞는 Learn 또는 Review 세션을 시작합니다.

개인 맞춤 학습 흐름

1

학습

Tutor는 코스 내용과 학생의 현재 진도를 바탕으로 다음 개념을 설명합니다.

2

연습

연습은 학생 답변에 맞춰 조정되며 필요할 때 힌트, 더 쉬운 예, 긍정적 피드백을 제공합니다.

3

오답 복습

의미 있는 오답은 오답 노트에 요약되어 이후 복습이 실제 약점을 겨냥하게 합니다.

API config 항목

  • Provider는 OpenAI 또는 SiliconFlow 같은 모델 서비스를 선택합니다.
  • Base URL은 Tutor1on1이 모델 서비스에 요청을 보내는 endpoint이며, 선택한 provider에 맞게 표시됩니다.
  • API Key는 로컬에 저장되며 tutor, TTS, STT 등 모델 서비스가 필요한 기능에 사용됩니다.
  • Test API key는 {baseUrl}/models를 호출해 key를 검증하고 사용 가능한 text, TTS, STT 모델 목록을 불러옵니다. 모델을 다운로드하지 않으며 새 모델을 자동 선택하지 않습니다.
  • Text model은 tutor 대화에 사용됩니다. Thinking effort는 reasoning control을 지원하는 provider에서만 표시됩니다.
  • TTS model은 음성 출력, STT model은 음성 입력 전사를 제어합니다. 음성을 쓰지 않으면 비워 둘 수 있습니다.
  • 저장된 API config는 provider URL, 선택한 모델, thinking effort, key hash를 저장해 검증된 설정을 다시 사용할 수 있게 합니다.
Marketplace

코스 찾기

교사는 코스 listing을 공개합니다. 학생은 탐색하고 수강 신청을 하며 승인 후 다운로드합니다.

Sessions

개인 피드백

Learn은 새 내용, Review는 회상 연습에 사용합니다. 피드백은 학생 답변에 맞춰 바뀌며 진도를 보이게 합니다.

Sync

연속성

교사와 학생 홈은 계정에 연결된 수강 상태, 세션, 진도, 오답 노트 신호를 동기화합니다.

자주 겪는 문제

  • 로그인이나 코스 로딩이 실패하면 앱이 api.tutor1on1.org에 연결되는지 확인하세요.
  • 학생이 코스에 들어갈 수 없으면 수강 신청이 승인되었고 bundle이 다운로드되었는지 확인하세요.
  • 모델 목록이 오래되어 보이면 현재 provider와 key로 Test API key를 다시 실행하세요.
  • 여러 기기의 진도가 오래되어 보이면 동기화 완료를 기다리거나 수동 새로고침을 실행하세요.
