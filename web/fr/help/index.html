Utiliser Tutor1on1 selon le rôle.

Les enseignants publient et gèrent les cours. Les élèves demandent une inscription, téléchargent le contenu approuvé et reçoivent un guidage, des exercices et une révision des erreurs adaptés à leur progression.

Parcours enseignant

Publier et personnaliser les cours.

  1. Connectez-vous comme enseignant ou créez un compte enseignant.
  2. Créez ou chargez une version locale du cours.
  3. Téléversez le bundle validé dans le marketplace.
  4. Approuvez les demandes d'inscription des élèves.
  5. Depuis la page enseignant, suivez la progression, les erreurs récurrentes et les sessions synchronisées de chaque élève.

Parcours élève

S’inscrire et étudier à son rythme.

  1. Connectez-vous comme élève ou créez un compte élève.
  2. Ouvrez le marketplace et demandez une inscription à un cours.
  3. Attendez l'approbation de l'enseignant.
  4. Téléchargez le bundle de cours approuvé.
  5. Ouvrez le cours et lancez une session Learn ou Review qui suit votre progression.

Boucle d’apprentissage personnalisé

1

Apprendre

Le tutor explique le concept suivant dans le contexte du cours et de la progression actuelle de l'élève.

2

Pratiquer

Les exercices s'adaptent aux réponses de l'élève, avec indices, exemples plus simples et feedback positif si nécessaire.

3

Réviser les erreurs

Les erreurs significatives sont résumées dans le carnet d’erreurs afin que la révision cible de vraies faiblesses.

Champs API config

  • Provider choisit le service de modèles, par exemple OpenAI ou SiliconFlow.
  • Base URL est l’endpoint utilisé par Tutor1on1 pour le provider sélectionné.
  • API Key est gardée localement et utilisée pour tutor, TTS et STT quand ces fonctions en ont besoin.
  • Test API key appelle {baseUrl}/models, vérifie la key et charge les listes de modèles text, TTS et STT disponibles. Il ne télécharge pas de modèle et ne sélectionne pas automatiquement un nouveau modèle.
  • Text model alimente la conversation tutor. Thinking effort apparaît seulement pour les providers qui prennent en charge les contrôles de reasoning.
  • TTS model contrôle la sortie vocale. STT model contrôle la transcription voix-texte. Ils peuvent rester vides si vous n’utilisez pas la voix.
  • Saved API configs conserve provider URL, modèles choisis, thinking effort et key hash pour réutiliser des configurations validées.

Utiliser ChatGPT OAuth au lieu d’une API key

  1. Ouvrez Settings, API Settings, puis choisissez OpenAI Codex (ChatGPT OAuth) comme Provider.
  2. Cliquez sur Login with ChatGPT et terminez la connexion dans le navigateur. Si l’app ne revient pas automatiquement, collez le redirect URL final ou l’authorization code dans la boîte de dialogue.
  3. Cliquez sur Refresh models, choisissez un text model comme gpt-5.5, puis enregistrez l’API config.
  4. Lancez ou rouvrez une session tutor et choisissez le modèle Codex OAuth enregistré dans le sélecteur de modèles.

Cette option utilise le quota ChatGPT/Codex OAuth pour les conversations texte du tutor. TTS et STT utilisent toujours des providers avec API key quand la voix est nécessaire.

Marketplace

Découverte des cours

Les enseignants publient des fiches de cours. Les élèves consultent, demandent une inscription et téléchargent après approbation.

Sessions

Feedback personnel

Learn sert au nouveau contenu et Review à la pratique de rappel. Le feedback s'adapte à la réponse de l'élève et rend la progression visible.

Sync

Continuité

Les pages enseignant et élève synchronisent inscriptions, sessions, progression et signaux du carnet d’erreurs liés au compte.

Problèmes fréquents

  • Si la connexion ou le chargement du cours échoue, vérifiez que l’app atteint api.tutor1on1.org.
  • Si un élève ne peut pas entrer dans un cours, confirmez que l’inscription est approuvée et que le bundle est téléchargé.
  • Si la liste des modèles semble ancienne, relancez Test API key pour le provider et la key actifs.
  • Si la progression semble ancienne entre appareils, attendez la fin de la synchronisation ou lancez une actualisation manuelle.
