快速开始

按角色使用 Tutor1on1。

老师发布并管理课程；学生申请报名、下载已批准内容，并获得符合个人进度的指导、练习和错题复习。

先安装 老师流程 学生流程 API config ChatGPT OAuth

老师流程

发布并个性化管理课程。

  1. 以老师身份登录，或注册老师账号。
  2. 创建或加载本地课程版本。
  3. 将已校验的 bundle 上传到 marketplace。
  4. 审批学生报名请求。
  5. 在老师主页查看每个学生的进度、常见错误和同步过来的会话。

学生流程

报名并按自己的节奏学习。

  1. 以学生身份登录，或注册学生账号。
  2. 打开 marketplace，申请报名课程。
  3. 等待老师批准。
  4. 下载已批准的课程 bundle。
  5. 打开课程，开始跟随个人进度的 Learn 或 Review 会话。

个性化学习闭环

1

学习

Tutor 会结合课程内容和学生当前进度，讲解下一步知识点。

2

练习

练习会根据学生回答调整，必要时给提示、更简单的例子和正反馈。

3

复习错题

有意义的错误会进入错题集，让后续复习真正针对薄弱点。

API config 里面的内容

  • 提供方选择模型服务，例如 OpenAI 或 SiliconFlow。
  • Base URL 是 Tutor1on1 请求模型服务的地址，应用会按当前提供方显示。
  • API Key 保存在本机，用于 tutor、TTS、STT 等需要模型服务的功能。
  • 测试 API key 会请求 {baseUrl}/models，验证 key，并加载可用的文本、TTS、STT 模型列表。它不会下载模型，也不会自动选择新模型。
  • 文本模型 用于对话式 tutor。Thinking effort 只在提供方支持 reasoning 控制时显示。
  • TTS 模型 控制语音输出；STT 模型 控制语音转文字。如果不用语音，可以留空。
  • 已保存的 API 配置 保存 provider URL、选择的模型、thinking effort 和 key hash，方便复用已验证配置。

不用 API key，改用 ChatGPT OAuth

  1. 打开设置里的 API Settings，把 Provider 选成 OpenAI Codex (ChatGPT OAuth)。
  2. 点击 Login with ChatGPT，在浏览器完成登录。如果应用没有自动返回，把最后的 redirect URL 或 authorization code 粘贴到弹窗里。
  3. 点击 Refresh models，选择文本模型，例如 gpt-5.5，然后保存 API 配置。
  4. 开始或重新打开 tutor 会话，在模型选择器里选择已保存的 Codex OAuth 模型。

这个选项使用 ChatGPT/Codex OAuth quota 来跑 tutor 文本对话。需要语音时，TTS 和 STT 仍然使用 API-key provider。

Marketplace

课程发现

老师发布课程展示；学生浏览课程、申请报名，并在批准后下载。

Sessions

个人反馈

Learn 用于新内容，Review 用于回忆练习。反馈会根据学生回答调整，并保持进步可见。

Sync

连续学习

老师和学生主页会同步已连接账号的报名、会话、进度和错题信号。

常见问题

  • 如果登录或课程加载失败，先确认应用能连到 api.tutor1on1.org。
  • 如果学生无法进入课程，先确认报名已批准，并且 bundle 已下载。
  • 如果模型列表看起来过期，对当前 provider 和 key 重新运行“测试 API key”。
  • 如果多设备进度不同步，等待同步完成，或手动刷新一次。
