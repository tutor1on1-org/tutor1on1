How to use Tutor1on1

Quick help for teachers and students.

Tutor1on1 is available on Windows, Android, and macOS. Teachers publish structured courses. Students enroll, download approved content, and study through guided sessions.

1. First launch

  1. Open the install guide and choose Windows, Android, or macOS.
  2. Choose a language if needed.
  3. Open Settings and configure the required API keys, then save them before you start using tutor features.
  4. Use the tabs on the welcome page to log in, register as a teacher, or register as a student.
  5. Teacher registration asks for profile fields and subject labels. Student registration is simpler.

Marketplace and courses

  • The marketplace entry is the small icon near the top-right corner of the app.
  • Teachers use it to work with remote course publishing and related marketplace flows.
  • Students use it to browse courses, request enrollment, and start the download flow after approval.

Teacher quick start

Publish and manage courses.

  1. Log in as a teacher.
  2. Create or load a local course version.
  3. Upload the validated bundle to the marketplace.
  4. Approve enrollment requests from students.
  5. Review student progress and synced sessions from the teacher home page.

Student quick start

Enroll and study.

  1. Log in as a student.
  2. Browse the marketplace and request enrollment in a course.
  3. Wait for teacher approval.
  4. Download the approved course bundle.
  5. Open the course and start a learning session.

2. Study session modes

Learn

Use Learn to work through new material with the tutor.

Review

Use Review to answer questions and strengthen recall on existing knowledge points.

Summary

Use Summary when you want the session to update the current mastery outcome for that topic.

3. Sync and continuity

  • Teacher and student home pages run sync in the background when the account is linked to the server.
  • Enrollment state, sessions, and progress can move across devices.
  • There is also a force-download option on the student side when you need to refresh from the server copy.

4. Common problems

  • If login or course loading fails, first check that the app can reach `api.tutor1on1.org`.
  • If a student cannot enter a course, confirm that enrollment was approved and the bundle was downloaded successfully.
  • If progress looks stale across devices, wait for sync to finish or run a manual refresh.
  • If a session is interrupted, reopen the same course and continue the existing session.
